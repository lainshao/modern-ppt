# Modern PPT 排版规范(投屏 deck)

> 这是 `modern-ppt` 的硬约束细则,生成/迭代前先按它 review。源自实战:从 0 做完一份 27 页投屏 deck、30+ 轮批注磨出来的。

## 1. 字号:单一 token + 层级编码

| 角色 | 投屏档 | 说明 |
|---|---|---|
| Display | `min(8.4vw,12vh)` ~120px | 封面/幕封/大字断言/巨号 KPI |
| 页标题 H2 | `min(5vw,5.6vh)` ~57px | 每页主标题(`.h-xl-zh`) |
| 卡片标题 H3 | `max(17px,1.5vw)` ~29px | 卡内小标题(`.sub-card .ttl`) |
| **正文 Body** | **`--fs-body` = `max(17px,min(1.1vw,1.95vh))` ~21px** | 所有描述/段落/列表/表格单元格,**统一走这一个 token** |
| Mono 角标 | 14px | 页眉/编号/标签/kicker(`.t-cat/.t-meta`) |

- **正文只允许 `var(--fs-body)`**,禁止组件各写各的 `max(16px,.94vw)`。改一个变量切场景:投屏 21px / 视频剪辑 32px(`max(18px,min(2.3vw,3vh))`)。
- 每页字号种类 ≤ 上表几档 + 大数字特例;**字号大小本身就编码层级**,层级不许倒挂。
- 内容放不下:先删字 / 拆页 / 换版式,**不靠缩小字号**。

## 2. 字体 / 强调

- 两套字体:无衬线主字(标题/正文/卡片)+ 等宽辅字(页码/meta/编号/URL)。不混第三种。
- **禁止斜体**(中文场景观感差);重点 = **黑色加粗**(weight 500-600)。
- accent 色(橙)/ 公司品牌色**只用于**:标题关键词、图表数据、1px 分隔线。**正文不上色。**

## 3. 标题层级:靠分隔线,不靠加大字号 ⭐

让一行字"读起来像标题",靠的是 **header 与内容之间有清晰分隔 + header 是安静 label 让内容唱主角**,**不是**把字号加大(加大反而和加粗的正文打架、更难读——实测翻过车)。
- 栏目标题用 `.col-head`:小号 ~18px semibold 灰字 + `border-bottom:2px solid var(--ink)` + 下方留白。和数据表的列表头是同一套。
- 层级仍要清楚:页标题 ＞ 栏目标题 ＞ 正文 ＞ 角标。

## 4. 对齐 / 连排

- 每页一个主对齐逻辑;信息/卡片/步骤页统一左对齐。
- **内容紧跟标题连排,禁止 `margin-top:auto` 锚底**(否则标题与内容大裂缝)。
- **编号列表(01/02/03)用一个统一 grid 共享列轨道 + `font-feature-settings:'tnum'`**;别每条各开 `auto 1fr`(比例字体下 "1"/"3" 宽度不同 → 各行标题左缘参差)。

## 5. 版心 / 安全区(1920×1080)

- 左右 ~96px(5vw)、顶 ~60px、**底部安全区 ≥6vh**——满高内容块(stack/卡片/bar 图)不要贴右下角导航提示 `#hint` 与分页点。
- 多列用明确 grid,不自由漂浮;同类型页标题位置一致。

## 6. 数据 / 卡片

- 强弱用 **5 格直角方块刻度**(领跑=5 满格、领先=4…),可横向扫读比较;别只写文字。
- 卡片**小标在上**(不用右上角标,长标题会重叠);**一组卡别随手单格高光**——高光要么有数据/焦点语义,要么统一无。
- KPI 大数字可作第 4 种特例字号,但一页只一个 hero 数字。

## 7. 颜色

- 一份 deck 只用**一个 accent**(默认安全橙 `#FF6B35`)。
- accent 不满屏铺(高饱和满屏刺眼):封面/幕封用**深墨底 + 关键词上色**;split 收束半屏橙可接受。
- 浅色高饱和 accent(黄/绿)`--accent-on` 必须用纯黑。

## 8. 命名 / 交付

- 文件名 = 主体名 + 最后更新日期 + version(如有多版),不带技术词。
- 默认产出后 `open` 在浏览器打开预览。

## 踩坑清单(别重踩)

1. **截图验证陷阱**:Chrome 后台标签冻结动画时钟 → 截图拍到"第一帧空白",**不是 bug**。验证动效先按 `B` 切静态(`__setLowPowerMode(true,{persist:false})`)再截。
2. **只用已验证组件**:模板未带 CSS 的花式版式直接用会崩。
3. **bar 图**:`.row-fill` 必须 `display:block`(inline span width 无效);`bar-grow` 动画终值要写回 inline(WAAPI width 动画结束会弹回起始值)——模板已修好,别改坏。
4. **脚本改单文件**:锚唯一串、改完数 section 数;定位串撞车会误删页。
